  1. 数据库技术发展内在驱动：
    1. 业务发展
    2. 场景创新
    3. 硬件与云计算的发展
  2. scale up 纵向扩展，scale out 水平扩展

分布式关系数据库的发展

  1. 分布式系统定义：
    分布式系统是一种其组件位于不同的联网计算机上的系统，然后通过互相传递消息来进行通信和协调。为了达到共同的目标，这些组件会互相作用。

  2. 分布式技术的主要挑战：

    1. 如何最大程度的实现分治
    2. 如何实现全局的一致性
    3. 如何进行故障与部分失效的容错
    4. 如何应对不可靠的网络与网络分区

  3. CAP理论：只能满足一致性（Consistency）、可用性（Availability）、分区容忍性（Partition Tolerance）中的两项。

    1. 一致性：所有节点在同一时间的数据完全一致。
    2. 可用性：服务在正常响应时间内的可用。
    3. 分区容忍性：分布式系统在遇到某节点或网络分区故障的时候仍然能够提供满足一致性或可用性的服务。

  4. NewSQL：原生分布式关系型数据库。NewSQL是一类关系数据库，它寻求为在线交易处理（OLTP）工作提供NoSQL系统的可扩展性，同时维护传统数据库系统的ACID保证。

TiDB整体概述

我们到底需要一个什么样的数据库

  1. 我们到底需要一个什么样的数据库
    1. Sclae-out 弹性横向扩展。从弹性角度看，颗粒度越小越好。常见的颗粒度有集群、Database、表、分表或者分区表
    2. 强一致与高可用。节点数量越多，保存的延迟越高。
    3. 标准SQL支持ACID事务
    4. 云原生
    5. 混合数据服务HTAP
    6. 兼容主流生态与协议
  2. 数据技术栈领域里常见的基础因素
    1. 数据模型：关系模型、文档模型、Key-Value模型等
    2. 数据存储与检索结构：B-tree、LSM-tree、分形树等
    3. 数据格式：结构化数据、非结构化数据、半结构化数据
    4. 存储引擎：InnoDB、RocksDB等
    5. 复制协议：Raft、Paxos等
    6. 分布式事务模型：XA、Percolator等
    7. 数据架构：Share-Disk、Share-Nothing、Share-Everything等
    8. 优化器算法
    9. 执行引擎
    10. 计算引擎

如何构建一个分布式存储系统

  1. 我们需要一个什么样的存储引擎
    1. 更细粒度的弹性扩缩容
    2. 高并发读写
    3. 数据不丢不错
    4. 多副本保障一致性及高可用
    5. 支持分布式事务
  2. 数据库核心：数据结构
    TiKV单节点选择了基于LSM-tree的RocksDB引擎
  3. 多版本控制（MVCC）
    TiKV的MVCC实现是通过在Key后面添加版本号来实现

新一代HTAP数据库选型

TiDB关键技术创新

  1. 自动分片技术是更细维度弹性的基础
    1. 全局有序的KV map
    2. 按照等长大小策略自动分片（96M）
    3. 每个分片是连续的KV，通过Start/End Key来寻址
    4. 每个分片seek成本固定
    5. 我们称该分片为Region，它是复制、调度的最小单位
  2. 弹性的分片构建成了动态的系统
    1. 自动merge
      • 96MB自增分片
      • 20MB合并分片
  3. Multi-Raft将复制组更离散
    • Raft、Multi-raft
    • leader、follower、learner
    • 目前是强主模式、读写在leader上
    • 4.0版本开启follower read
  4. 基于Multi-Raft实现写入的线性扩展
    • 当我们新增一个物理节点时，也就意味着整个集群的写入容量会进行线性增长。
  5. 基于Multi-Raft实现跨IDC单表多节点写入
  6. 去中心化的分布式事务
  7. Local Read and Geo-partition
    1. 多地部署支持，低访问延时。
    2. 数据安全合规，符合数据不出境场景。
    3. 支持异地多活容灾。
    4. 支持冷热数据分离。
  8. 更大数据容量下的TP与AP融合
    • TiDB引入了实时更新的列式引擎，即解决了资源隔离，又提升了AP效率。
    • 在列存上引入MPP模型，实现了SQL join的下推与并行处理。
    • 通过Raft-base replication实现了更时效性。
    • 融合大数据生态，比如TiSpark。
  9. 数据服务的统一
